Although the admin portal is not complex and supposedly easy to use, this documentation page still exists just in case.

ID's

The first confusing concept might be the use of ID's that this blog employs in order to create elements; and this is compulsory. The form might not force you to enter an id but it does indeed store the empty space as an ID.

ID's are important for when you want to delete posts as posts are deleted used their id and so please make your id's easy to remember. Despite this warning, there is support for remembering ID's as you might notice a little badge on each post with the post id on it.

The naming convention for creating IDs is in this format: eg 1F, 2DE, 74E. These are all examples on the preferred ways to name IDs.

Deleting Posts

We've gone over ID's in the previous heading and so this should be fairly easy to understand. All you have to do is to select the corresponding id of a post and click on delete post. Note that deletion is permanent so double check before deleting any post!

For any other complaints, please contact the developer.
